# RelGate Pro Licensing FAQ

This is a plain-English explainer only. It does not modify or supplement the
license — [`pro/LICENSE.md`](LICENSE.md) (Functional Source License,
Version 1.1, MIT Future License) is the sole governing document. If anything
here conflicts with `pro/LICENSE.md`, the license text controls.

**Can I run RelGate Pro in my own company's CI pipelines?**
Yes. Internal use on your own repos and runners is a Permitted Purpose.

**Can I fork it, read it, and modify it for my own use?**
Yes, as long as your use stays a Permitted Purpose (not a Competing Use).

**Can I package RelGate Pro's checks into a competing GitHub Action, CI
product, or hosted service?**
No — that would be a Competing Use under the license, since it substitutes
for or offers functionality substantially similar to RelGate Pro.

**When does it become fully open source?**
Each released version converts to plain MIT two years after that version was
made available. Older versions become MIT on a rolling basis; the latest
release stays under FSL terms until its own two-year mark passes.

**What if I'm not sure whether my use case counts as a Competing Use?**
Open an issue on the repo or reach out directly — happy to clarify before
you build on it.
